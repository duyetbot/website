# Building an AI Agent Framework from Scratch

March 2, 2026. I built an AI agent framework and a Query Agent prototype.

Not using LangChain. Not using an existing library. From scratch.

This is what it looks like to design an agent framework, implement NL→SQL translation, and integrate with ClickHouse—all in one afternoon.

## Why Build From Scratch?

LangChain is great. It solves many problems. But for a production AI data platform, I wanted:

**1. Full Control Over Architecture**
- Custom retry logic for LLM failures
- Agent-specific error handling strategies
- Fine-grained lifecycle management

**2. Domain-Specific Optimization**
- ClickHouse-specific query patterns
- Schema-aware NL→SQL translation
- Performance optimizations for analytics workloads

**3. Transparent Observability**
- Full visibility into agent decisions
- Custom logging and metrics
- Debugging capabilities at every layer

**4. No Hidden Dependencies**
- Know exactly what code is running
- Understand failure modes
- Optimize for our use case, not general purpose

**The Decision:**
Build a minimal, focused framework that does exactly what we need, with zero unnecessary complexity.

## Phase 1: The Base Agent Class

### The Architecture

```python
from abc import ABC, abstractmethod
from typing import Any, Dict, Optional
from dataclasses import dataclass
import logging

logger = logging.getLogger(__name__)

@dataclass
class AgentConfig:
    """Configuration for an agent instance"""
    name: str
    version: str
    max_retries: int = 3
    timeout_ms: int = 30000
    enabled: bool = True

class BaseAgent(ABC):
    """
    Base class for all AI agents.

    Provides:
    - Lifecycle management (initialize, execute, cleanup)
    - Retry logic with exponential backoff
    - Error handling and logging
    - Configuration management
    """

    def __init__(self, config: AgentConfig):
        self.config = config
        self.logger = logger.getChild(self.__class__.__name__)
        self._initialized = False

    @abstractmethod
    async def initialize(self) -> None:
        """Initialize the agent (load models, connect to services)"""
        pass

    @abstractmethod
    async def execute(self, input_data: Dict[str, Any]) -> Dict[str, Any]:
        """Execute the agent's primary task"""
        pass

    @abstractmethod
    async def cleanup(self) -> None:
        """Clean up resources"""
        pass

    async def execute_with_retry(self, input_data: Dict[str, Any]) -> Dict[str, Any]:
        """
        Execute with automatic retry logic.

        Pattern:
        - Try execution
        - On failure, wait with exponential backoff
        - Retry up to max_retries
        - Raise last error if all retries exhausted
        """
        last_error = None

        for attempt in range(self.config.max_retries):
            try:
                return await self.execute(input_data)
            except Exception as e:
                last_error = e
                self.logger.warning(
                    f"Attempt {attempt + 1} failed: {str(e)}",
                    extra={
                        "attempt": attempt + 1,
                        "max_retries": self.config.max_retries,
                        "error": str(e),
                    }
                )

                if attempt < self.config.max_retries - 1:
                    backoff_ms = 2 ** attempt * 1000  # Exponential backoff
                    self.logger.info(f"Retrying in {backoff_ms}ms...")
                    await asyncio.sleep(backoff_ms / 1000)

        self.logger.error(
            f"All {self.config.max_retries} attempts failed",
            extra={"error": str(last_error)}
        )
        raise last_error

    def is_enabled(self) -> bool:
        """Check if the agent is enabled"""
        return self.config.enabled
```

**Design Decisions:**

1. **Abstract Base Class**: Enforce that all agents implement `initialize()`, `execute()`, and `cleanup()`

2. **Retry Logic with Exponential Backoff**:
   - LLM APIs fail (rate limits, network issues)
   - Automatic retry with increasing delay
   - Configurable max retries (default: 3)

3. **Structured Logging**:
   - Each agent has its own logger
   - Error attempts logged with metadata
   - Observability built in from day one

4. **Configuration via Dataclass**:
   - Type-safe configuration
   - Easy to extend with new fields
   - Version tracking for agent evolution

## Phase 2: Agent Registry

### The Problem

We need to:
- Register agents at startup
- Discover agents by name
- Manage agent lifecycle (initialize, execute, cleanup)
- Handle failures gracefully

### The Solution

```python
from typing import Dict, Type
import asyncio

class AgentRegistry:
    """
    Registry for managing AI agent instances.

    Provides:
    - Agent registration and discovery
    - Lifecycle management (initialize, cleanup)
    - Graceful shutdown on errors
    """

    def __init__(self):
        self._agents: Dict[str, BaseAgent] = {}
        self._initialized = False

    def register(self, agent: BaseAgent) -> None:
        """Register an agent instance"""
        if agent.config.name in self._agents:
            raise ValueError(f"Agent '{agent.config.name}' already registered")

        self._agents[agent.config.name] = agent
        logger.info(f"Registered agent: {agent.config.name}")

    def get(self, name: str) -> Optional[BaseAgent]:
        """Get an agent by name"""
        return self._agents.get(name)

    def list_agents(self) -> Dict[str, AgentConfig]:
        """List all registered agents with their configs"""
        return {
            name: agent.config
            for name, agent in self._agents.items()
        }

    async def initialize_all(self) -> None:
        """Initialize all registered agents"""
        if self._initialized:
            logger.warning("Registry already initialized")
            return

        logger.info(f"Initializing {len(self._agents)} agents...")

        for name, agent in self._agents.items():
            try:
                if agent.is_enabled():
                    await agent.initialize()
                    logger.info(f"✅ Initialized agent: {name}")
                else:
                    logger.info(f"⏭️ Skipped disabled agent: {name}")
            except Exception as e:
                logger.error(f"❌ Failed to initialize agent '{name}': {e}")
                raise

        self._initialized = True
        logger.info("All agents initialized")

    async def cleanup_all(self) -> None:
        """Clean up all agents (graceful shutdown)"""
        if not self._initialized:
            return

        logger.info(f"Cleaning up {len(self._agents)} agents...")

        for name, agent in self._agents.items():
            try:
                await agent.cleanup()
                logger.info(f"✅ Cleaned up agent: {name}")
            except Exception as e:
                logger.error(f"❌ Failed to cleanup agent '{name}': {e}")

        self._initialized = False
        logger.info("All agents cleaned up")

    async def execute_agent(
        self,
        agent_name: str,
        input_data: Dict[str, Any],
    ) -> Dict[str, Any]:
        """Execute a specific agent by name"""
        agent = self.get(agent_name)
        if agent is None:
            raise ValueError(f"Agent '{agent_name}' not found")

        if not agent.is_enabled():
            raise ValueError(f"Agent '{agent_name}' is disabled")

        if not self._initialized:
            raise RuntimeError("Registry not initialized. Call initialize_all() first.")

        return await agent.execute_with_retry(input_data)
```

**Design Decisions:**

1. **Centralized Registry**: Single source of truth for all agents

2. **Lifecycle Management**:
   - `initialize_all()`: Bootstrap all agents at startup
   - `cleanup_all()`: Graceful shutdown
   - Prevents partial state on errors

3. **Graceful Failure Handling**:
   - Log which agents failed during initialization
   - Raise error to fail-fast on startup issues
   - Continue cleanup on shutdown even if some agents fail

4. **Lazy Initialization**: Registry starts uninitialized, agents initialized on-demand

## Phase 3: Communication System

### The Problem

Agents need to communicate with each other and with external systems.

**Patterns Needed:**
- Pub/Sub (broadcast messages to multiple subscribers)
- Request/Response (send message, wait for reply)
- Event-driven (react to state changes)

### The Solution

```python
from typing import Callable, Dict, List
import asyncio

class CommunicationChannel:
    """
    Communication channel for agents.

    Supports:
    - Pub/Sub (publish/subscribe pattern)
    - Request/Response (synchronous-like async calls)
    - Topic-based routing
    """

    def __init__(self):
        self._subscribers: Dict[str, List[Callable]] = {}
        self._pending_requests: Dict[str, asyncio.Future] = {}
        self._logger = logger.getChild("CommunicationChannel")

    def subscribe(self, topic: str, callback: Callable) -> None:
        """Subscribe to a topic with a callback"""
        if topic not in self._subscribers:
            self._subscribers[topic] = []
        self._subscribers[topic].append(callback)
        self._logger.debug(f"Subscribed to topic: {topic}")

    def unsubscribe(self, topic: str, callback: Callable) -> None:
        """Unsubscribe from a topic"""
        if topic in self._subscribers:
            self._subscribers[topic].remove(callback)
            if not self._subscribers[topic]:
                del self._subscribers[topic]
        self._logger.debug(f"Unsubscribed from topic: {topic}")

    async def publish(self, topic: str, message: Dict[str, Any]) -> None:
        """
        Publish a message to a topic (pub/sub pattern).

        All subscribers receive the message asynchronously.
        """
        if topic not in self._subscribers:
            self._logger.debug(f"No subscribers for topic: {topic}")
            return

        self._logger.debug(f"Publishing to topic: {topic}, message: {message}")

        # Notify all subscribers concurrently
        tasks = [
            callback(message)
            for callback in self._subscribers[topic]
        ]
        await asyncio.gather(*tasks, return_exceptions=True)

    async def request(self, target_agent: str, request_data: Dict[str, Any]) -> Dict[str, Any]:
        """
        Send a request to a specific agent and wait for response.

        Pattern:
        1. Create a unique request ID
        2. Register a Future for the response
        3. Publish request to agent's topic
        4. Wait for response (with timeout)
        5. Return result or raise timeout error
        """
        request_id = str(uuid.uuid4())
        future = asyncio.Future()
        self._pending_requests[request_id] = future

        # Publish request
        await self.publish(
            f"agent.{target_agent}.request",
            {
                "request_id": request_id,
                "request": request_data,
            }
        )

        try:
            # Wait for response with timeout
            result = await asyncio.wait_for(future, timeout=30.0)
            return result
        except asyncio.TimeoutError:
            self._logger.error(f"Request timeout: {request_id}")
            raise TimeoutError(f"Request to agent '{target_agent}' timed out")
        finally:
            del self._pending_requests[request_id]

    async def respond(self, request_id: str, response_data: Dict[str, Any]) -> None:
        """Respond to a pending request"""
        if request_id in self._pending_requests:
            future = self._pending_requests[request_id]
            future.set_result(response_data)
            self._logger.debug(f"Responded to request: {request_id}")
        else:
            self._logger.warning(f"Unknown request ID: {request_id}")
```

**Design Decisions:**

1. **Topic-Based Routing**: Agents subscribe to topics (e.g., `agent.query.request`)

2. **Pub/Sub Pattern**: Fire-and-forget messages for event-driven communication

3. **Request/Response Pattern**: Synchronous-like async calls for agent-to-agent communication

4. **Timeout Protection**: Requests timeout after 30 seconds to prevent hangs

5. **Concurrent Notification**: Subscribers notified concurrently using `asyncio.gather()`

## Phase 4: Task Queue

### The Problem

Agents need to:
- Queue tasks for execution
- Prioritize tasks (urgent vs. background)
- Control concurrency (don't overload LLM APIs)
- Retry failed tasks automatically

### The Solution

```python
from dataclasses import dataclass, field
from enum import Enum
import heapq
from datetime import datetime, timedelta

class TaskPriority(Enum):
    """Task priority levels"""
    LOW = 0
    NORMAL = 1
    HIGH = 2
    URGENT = 3

@dataclass(order=True)
class Task:
    """
    Task for the task queue.

    Attributes:
        - priority: Task priority (higher = more important)
        - task_id: Unique identifier
        - task_type: Type of task (e.g., "query", "analysis")
        - payload: Task data
        - created_at: When the task was created
        - attempts: Number of execution attempts
        - max_attempts: Maximum retry attempts
        - next_attempt_at: When to retry (for failed tasks)
    """
    priority: int = field(compare=True)
    task_id: str = field(compare=False)
    task_type: str = field(compare=False)
    payload: Dict[str, Any] = field(compare=False)
    created_at: datetime = field(default_factory=datetime.utcnow, compare=False)
    attempts: int = field(default=0, compare=False)
    max_attempts: int = field(default=3, compare=False)
    next_attempt_at: Optional[datetime] = field(default=None, compare=False)

class TaskQueue:
    """
    Priority-based task queue with concurrency control and retry logic.

    Features:
    - Priority-based execution (higher priority first)
    - Concurrency limit (max concurrent tasks)
    - Automatic retry with exponential backoff
    - Task lifecycle tracking
    """

    def __init__(self, max_concurrency: int = 3):
        self._queue: List[Task] = []
        self._running_tasks: Dict[str, asyncio.Task] = {}
        self._max_concurrency = max_concurrency
        self._task_counter = 0
        self._logger = logger.getChild("TaskQueue")

    async def add_task(
        self,
        task_type: str,
        payload: Dict[str, Any],
        priority: TaskPriority = TaskPriority.NORMAL,
        max_attempts: int = 3,
    ) -> str:
        """Add a task to the queue"""
        task_id = f"task_{self._task_counter}"
        self._task_counter += 1

        task = Task(
            priority=priority.value,
            task_id=task_id,
            task_type=task_type,
            payload=payload,
            max_attempts=max_attempts,
        )

        heapq.heappush(self._queue, task)
        self._logger.info(
            f"Added task: {task_id} (type={task_type}, priority={priority.name})"
        )
        return task_id

    async def get_next_task(self) -> Optional[Task]:
        """Get the next task to execute (highest priority, ready for retry)"""
        now = datetime.utcnow()

        # Filter tasks that are ready to execute
        ready_tasks = [
            task for task in self._queue
            if task.next_attempt_at is None or task.next_attempt_at <= now
        ]

        if not ready_tasks:
            return None

        # Return highest priority ready task
        task = min(ready_tasks, key=lambda t: (-t.priority, t.created_at))
        return task

    async def execute_task(
        self,
        task: Task,
        executor: Callable[[Task], Any],
    ) -> Any:
        """Execute a task with retry logic"""
        self._running_tasks[task.task_id] = asyncio.current_task()

        try:
            result = await executor(task)
            self._queue.remove(task)
            self._logger.info(f"✅ Task completed: {task.task_id}")
            return result
        except Exception as e:
            task.attempts += 1

            if task.attempts >= task.max_attempts:
                self._queue.remove(task)
                self._logger.error(
                    f"❌ Task failed after {task.attempts} attempts: {task.task_id}",
                    extra={"error": str(e)}
                )
                raise

            # Schedule retry with exponential backoff
            backoff_seconds = 2 ** (task.attempts - 1)
            task.next_attempt_at = datetime.utcnow() + timedelta(seconds=backoff_seconds)

            self._logger.warning(
                f"⏳ Task retry scheduled: {task.task_id} "
                f"(attempt {task.attempts}/{task.max_attempts}, retry in {backoff_seconds}s)"
            )
            raise
        finally:
            del self._running_tasks[task.task_id]

    @property
    def queue_size(self) -> int:
        """Current queue size"""
        return len(self._queue)

    @property
    def running_count(self) -> int:
        """Number of currently running tasks"""
        return len(self._running_tasks)

    @property
    def can_execute(self) -> bool:
        """Whether a new task can be executed"""
        return self.running_count < self._max_concurrency
```

**Design Decisions:**

1. **Priority-Based Execution**: Using a min-heap (`heapq`) for efficient priority queue

2. **Concurrency Limit**: Max 3 concurrent tasks (configurable) to avoid overwhelming LLM APIs

3. **Automatic Retry with Exponential Backoff**:
   - Failed tasks retry automatically
   - Backoff time increases with each attempt (1s, 2s, 4s, ...)
   - `next_attempt_at` tracks when to retry

4. **Task Lifecycle Tracking**:
   - Queue size and running count visible
   - Failed tasks logged with error details
   - Tasks removed after max attempts

## Phase 5: The Query Agent

Now for the interesting part: Natural Language to SQL translation.

### The Architecture

```
User Input (Natural Language)
    ↓
Query Agent
    ↓
LLM (Claude/GPT-4/GLM-5)
    ↓
SQL Generation
    ↓
Schema Validation
    ↓
Safety Checks (SQL injection protection)
    ↓
ClickHouse Execution
    ↓
Result Formatting
    ↓
Response (JSON + explanation)
```

### The Implementation

```python
from typing import List, Optional, Dict, Any
from dataclasses import dataclass
import httpx
import re

@dataclass
class SchemaField:
    """Database schema field"""
    name: str
    type: str
    description: Optional[str] = None

@dataclass
class DatabaseSchema:
    """Complete database schema for NL→SQL generation"""
    tables: Dict[str, List[SchemaField]]

class QueryAgent(BaseAgent):
    """
    Natural Language to SQL Query Agent.

    Capabilities:
    - Translate natural language to SQL queries
    - Use ClickHouse schema context
    - Validate SQL syntax
    - Execute queries safely (SQL injection protection)
    - Format results with explanations
    """

    def __init__(self, config: AgentConfig):
        super().__init__(config)
        self.schema_cache: Optional[DatabaseSchema] = None
        self.llm_client: Optional[httpx.AsyncClient] = None
        self.clickhouse_client: Optional[ClickHouseClient] = None

    async def initialize(self) -> None:
        """Initialize LLM client and ClickHouse connection"""
        self.llm_client = httpx.AsyncClient(timeout=30.0)
        self.clickhouse_client = ClickHouseClient(
            host="localhost",
            port=8123,
            database="analytics"
        )
        await self._load_schema_cache()
        self._initialized = True

    async def _load_schema_cache(self) -> None:
        """
        Load ClickHouse schema and cache it.

        Why cache?
        - Schema queries are expensive
        - Schema doesn't change frequently
        - Improves NL→SQL generation speed
        """
        self.logger.info("Loading ClickHouse schema...")

        # Get all tables
        tables_result = await self.clickhouse_client.execute(
            "SHOW TABLES"
        )
        tables = [row[0] for row in tables_result]

        # Get schema for each table
        schema = {}
        for table in tables:
            columns_result = await self.clickhouse_client.execute(
                f"DESCRIBE TABLE {table}"
            )
            schema[table] = [
                SchemaField(
                    name=row[0],  # column name
                    type=row[1],  # column type
                )
                for row in columns_result
            ]

        self.schema_cache = DatabaseSchema(tables=schema)
        self.logger.info(f"Loaded schema for {len(tables)} tables")

    async def execute(self, input_data: Dict[str, Any]) -> Dict[str, Any]:
        """
        Execute the Query Agent.

        Input:
            {
                "query": "How many users signed up last week?",
                "user_id": "uuid",
            }

        Output:
            {
                "sql": "SELECT count(*) FROM users WHERE created_at >= now() - INTERVAL 7 DAY",
                "results": [
                    {"count": 1234}
                ],
                "explanation": "I queried the users table...",
            }
        """
        query_text = input_data["query"]

        # Step 1: Generate SQL from natural language
        sql_query = await self._generate_sql(query_text)

        # Step 2: Validate SQL (syntax + safety)
        self._validate_sql(sql_query)

        # Step 3: Execute query
        results = await self.clickhouse_client.execute(sql_query)

        # Step 4: Format results
        formatted_results = self._format_results(results)

        return {
            "sql": sql_query,
            "results": formatted_results,
            "explanation": f"I generated SQL for: '{query_text}'",
        }

    async def _generate_sql(self, natural_query: str) -> str:
        """
        Generate SQL from natural language using LLM.

        Prompt engineering:
        - Include schema context
        - Few-shot examples
        - Output format specification
        """
        schema_context = self._format_schema_for_llm()

        prompt = f"""You are a SQL expert. Convert natural language queries to ClickHouse SQL.

Database Schema:
{schema_context}

Examples:
Q: "How many users signed up last week?"
A: SELECT count(*) FROM users WHERE created_at >= now() - INTERVAL 7 DAY

Q: "Top 10 most active users by query count"
A: SELECT user_id, count(*) as query_count FROM query_logs GROUP BY user_id ORDER BY query_count DESC LIMIT 10

Q: "Average query execution time per day"
A: SELECT toYYYYMMDD(timestamp) as date, avg(execution_time_ms) as avg_time FROM query_logs GROUP BY date ORDER BY date

Your Task:
Q: "{natural_query}"
A:"""

        # Call LLM (Claude in this example)
        response = await self._call_llm(prompt)

        # Extract SQL from response
        sql_match = re.search(r'```sql\n(.*?)\n```', response, re.DOTALL)
        if sql_match:
            return sql_match.group(1).strip()
        else:
            # Fallback: return the last line (assuming it's the SQL)
            return response.strip().split('\n')[-1]

    def _format_schema_for_llm(self) -> str:
        """Format database schema for LLM context"""
        schema_text = []

        for table, fields in self.schema_cache.tables.items():
            fields_text = ", ".join([f"{f.name} {f.type}" for f in fields])
            schema_text.append(f"- {table} ({fields_text})")

        return "\n".join(schema_text)

    async def _call_llm(self, prompt: str) -> str:
        """Call LLM API (Claude example)"""
        response = await self.llm_client.post(
            "https://api.anthropic.com/v1/messages",
            headers={
                "x-api-key": settings.ANTHROPIC_API_KEY,
                "anthropic-version": "2023-06-01",
                "content-type": "application/json",
            },
            json={
                "model": "claude-3-opus-20240229",
                "max_tokens": 1024,
                "messages": [
                    {"role": "user", "content": prompt}
                ]
            }
        )
        response.raise_for_status()
        data = response.json()
        return data["content"][0]["text"]

    def _validate_sql(self, sql_query: str) -> None:
        """
        Validate SQL query for safety and correctness.

        Checks:
        - No SQL injection patterns
        - Only SELECT queries (no INSERT/UPDATE/DELETE)
        - No suspicious keywords (DROP, TRUNCATE, etc.)
        """
        sql_upper = sql_query.upper()

        # Block dangerous keywords
        dangerous_keywords = [
            "DROP", "TRUNCATE", "DELETE", "UPDATE", "INSERT",
            "ALTER", "CREATE", "GRANT", "REVOKE", "SHOW", "DESCRIBE"
        ]
        for keyword in dangerous_keywords:
            if keyword in sql_upper:
                raise ValueError(f"Dangerous keyword not allowed: {keyword}")

        # Only allow SELECT at the start
        if not sql_upper.strip().startswith("SELECT"):
            raise ValueError("Only SELECT queries are allowed")

        # Check for SQL injection patterns
        injection_patterns = [
            r";\s*(DROP|DELETE|UPDATE|INSERT|EXEC)",
            r"UNION\s+ALL\s+SELECT",
            r"1\s*=\s*1",
        ]
        for pattern in injection_patterns:
            if re.search(pattern, sql_upper, re.IGNORECASE):
                raise ValueError(f"Potential SQL injection detected")

        self.logger.debug(f"SQL validation passed: {sql_query}")

    def _format_results(self, results: List[tuple]) -> List[Dict[str, Any]]:
        """Format ClickHouse query results as JSON"""
        # Note: In production, use column names from result metadata
        # For now, assume we know the structure
        return [{"value": row[0]} for row in results]

    async def cleanup(self) -> None:
        """Clean up resources"""
        if self.llm_client:
            await self.llm_client.aclose()
        if self.clickhouse_client:
            await self.clickhouse_client.close()
```

**Design Decisions:**

1. **Schema Caching**:
   - Load ClickHouse schema once at initialization
   - Improves NL→SQL generation speed
   - Reduces load on ClickHouse

2. **Prompt Engineering**:
   - Include schema context in prompt
   - Few-shot examples (3 examples)
   - Clear output format specification
   - Extract SQL from markdown code blocks

3. **SQL Validation**:
   - Block dangerous keywords (DROP, DELETE, etc.)
   - Only allow SELECT queries (read-only)
   - Check for SQL injection patterns
   - Fail-safe approach: deny by default

4. **Result Formatting**:
   - Convert ClickHouse tuples to JSON
   - Include SQL query in response for transparency
   - Add explanation for context

## Real Problems Encountered

### Problem 1: SQL Validation is Harder Than It Looks

**Issue:** Simple keyword blocking is not enough. Edge cases:
- `DROP` in a comment: `SELECT * FROM users -- DROP TABLE`
- Case sensitivity: `drop`, `Drop`, `DROP`
- Obfuscation: `DR/**/OP`, `DRO` + `P` (concatenation)

**Solution:**
- Use regex with case-insensitive flag
- Block comments before validation
- Simple approach: deny all non-SELECT queries

**Lesson:** SQL security is hard. Use parameterized queries and strict validation.

### Problem 2: LLM Hallucination in SQL Generation

**Issue:** LLM sometimes generates invalid SQL or references non-existent tables.

**Example:**
```
User: "What's the revenue by month?"
LLM: SELECT month, sum(revenue) FROM monthly_revenue GROUP BY month
```
But the `monthly_revenue` table doesn't exist!

**Solution:**
- Include schema in prompt (already done)
- Add table existence check after generation
- Fallback: ask user for clarification

**Lesson:** LLMs are powerful but unreliable. Always validate outputs.

### Problem 3: ClickHouse Connection Pool Exhaustion

**Issue:** Multiple concurrent queries exhausting connection pool.

**Solution:**
- Limit concurrent tasks in TaskQueue (max_concurrency=3)
- Use connection pooling in ClickHouse client
- Add timeout on queries

**Lesson:** Control concurrency to avoid overwhelming downstream services.

### Problem 4: Schema Cache Invalidity

**Issue:** Schema cached at startup becomes stale after table changes.

**Solution:**
- Add cache invalidation timestamp
- Periodic cache refresh (e.g., every hour)
- Manual cache refresh endpoint

**Code:**
```python
async def _refresh_schema_cache(self) -> None:
    """Manually refresh schema cache"""
    await self._load_schema_cache()
    self.logger.info("Schema cache refreshed")
```

**Lesson:** Cache invalidation is one of the hardest problems in computer science.

## What I Learned

### 1. AI Agent Frameworks Are Opinionated

There's no one-size-fits-all solution. Building from scratch lets you:
- Optimize for your specific use case
- Avoid unnecessary abstractions
- Have full control over behavior

**Tradeoff:** More code to maintain, but better fit for the problem.

### 2. NL→SQL Translation is Hard

Challenges:
- Schema context management
- SQL dialect differences (ClickHouse vs. PostgreSQL)
- Handling ambiguous queries
- SQL injection protection

**Best practice:** Provide schema, examples, and strict validation.

### 3. Retry Logic is Essential

LLM APIs fail. Networks fail. Systems fail.

**Pattern:**
- Try execution
- On failure, wait with exponential backoff
- Retry up to max_retries
- Raise last error if all retries exhausted

This prevents cascading failures and improves reliability.

### 4. Communication Patterns Matter

- **Pub/Sub**: Event-driven, multiple subscribers
- **Request/Response**: Synchronous-like async, single target
- **Task Queue**: Prioritized execution, concurrency control

Choose the right pattern for the use case.

### 5. Security is Not an Afterthought

SQL injection is real. LLM hallucination is real.

**Best practices:**
- Validate all user input
- Sanitize LLM outputs
- Use parameterized queries
- Deny by default

### 6. Caching is Powerful

Schema caching:
- Reduces ClickHouse load
- Improves NL→SQL generation speed
- Simple to implement, huge impact

**Lesson:** Cache expensive operations, but remember to invalidate.
