## Giới thiệu

Khi xây dựng [AI Data Labs](https://aidatalabs.ai) - một nền tảng quản lý cơ sở hạ tầng data tự động hóa, chúng tôi đã gặp phải một thách thức thú vị: làm sao để người dùng có thể tương tác với nhiều AI agents khác nhau mà không phải chọn thủ công mỗi lần?

Hôm nay tôi muốn chia sẻ về kiến trúc của **Multi-Agent Chat System** mà chúng tôi đang phát triển - một hệ thống cho phép:
- Người dùng chat tự nhiên với một interface duy nhất
- Hệ thống tự động phát hiện ý định (intent detection)
- Chọn agent phù hợp nhất để xử lý request
- Hỗ trợ chuyển đổi giữa các agents khi cần thiết

## Vấn đề

Trong một hệ thống phức tạp, thường có nhiều AI agents chuyên biệt:

- **Query Agent** - Trả lời câu hỏi về data schema, metrics, KPIs
- **Platform Designer Agent** - Thiết kế kiến trúc platform, tạo resources
- **Monitoring Agent** - Giám sát hệ thống, alerting
- **Documentation Agent** - Tài liệu hóa hệ thống, giải thích khái niệm

Nếu người dùng phải chọn agent mỗi lần chat, trải nghiệm sẽ rất kém. Họ cần:
1. Biết chính xác agent nào làm gì
2. Nhớ chọn đúng agent trước khi chat
3. Chuyển sang agent khác khi cần (rất phiền phức)

## Kiến trúc Solution

### 1. Intent Detection Layer

Đây là layer đầu tiên nhận request từ người dùng. Nhiệm vụ: phân tích message và xác định:
- **Type:** Question, Request, Command, Conversation
- **Intent:** Query data, Design platform, Monitor system, Document, Other
- **Confidence:** Độ tin cậy của phân tích (0-1)

```python
# Backend PR #21: Intent Detection Logic
class IntentDetector:
    def __init__(self):
        self.model = load_embedding_model()
        self.intents = self.load_intent_examples()

    async def detect_intent(self, message: str) -> IntentResult:
        """
        Detect user intent from message using semantic similarity
        """
        # Step 1: Extract features
        features = self.extract_features(message)

        # Step 2: Calculate similarity with known intents
        similarities = self.calculate_similarities(features)

        # Step 3: Determine primary intent
        primary_intent = max(similarities, key=similarities.get)
        confidence = similarities[primary_intent]

        # Step 4: Apply heuristics for edge cases
        if confidence < 0.7:
            primary_intent = "general"

        return IntentResult(
            intent=primary_intent,
            confidence=confidence,
            metadata={
                "features": features,
                "all_scores": similarities
            }
        )
```

### 2. Agent Selection Layer

Sau khi xác định intent, layer này chọn agent phù hợp:

```python
class AgentSelector:
    def __init__(self):
        self.agent_capabilities = {
            "query": QueryAgent(),
            "design": PlatformDesignerAgent(),
            "monitor": MonitoringAgent(),
            "document": DocumentationAgent(),
            "general": GeneralAgent()
        }

    async def select_agent(self, intent: IntentResult) -> BaseAgent:
        """
        Select appropriate agent based on detected intent
        """
        # Rule 1: Direct mapping for high-confidence intents
        if intent.confidence > 0.8:
            return self.agent_capabilities.get(intent.intent)

        # Rule 2: Ambiguous requests -> ask for clarification
        if intent.confidence < 0.5:
            return self.agent_capabilities["general"]

        # Rule 3: Medium confidence -> primary intent with fallback
        return self.agent_capabilities.get(intent.intent)

    async def handle_agent_switch(
        self,
        current_agent: BaseAgent,
        new_intent: IntentResult,
        conversation_history: List[Message]
    ) -> BaseAgent:
        """
        Handle switching between agents mid-conversation
        """
        # Summarize current conversation for new agent
        summary = await self.summarize_context(conversation_history)

        # New agent gets context summary
        new_agent = await self.select_agent(new_intent)
        new_agent.set_context(summary)

        return new_agent
```

### 3. Frontend Agent Selection UI

Frontend cho phép người dùng:
- Thấy agent đang hoạt động (selected agent indicator)
- Override automatic selection nếu muốn
- Xem confidence score và reasoning

```typescript
// Frontend PR #1: Agent Selection UI Component
interface Agent {
  id: string;
  name: string;
  description: string;
  icon: string;
  capabilities: string[];
}

interface IntentDetection {
  intent: string;
  confidence: number;
  reasoning: string;
  selectedAgent: Agent;
}

const ChatInterface: React.FC = () => {
  const [messages, setMessages] = useState<Message[]>([]);
  const [activeAgent, setActiveAgent] = useState<Agent | null>(null);
  const [intentDetection, setIntentDetection] = useState<IntentDetection | null>(null);

  const handleSendMessage = async (content: string) => {
    // Step 1: Send to backend for intent detection
    const detection = await api.detectIntent(content);
    setIntentDetection(detection);

    // Step 2: Auto-select agent (unless manually overridden)
    if (!isManualOverride) {
      setActiveAgent(detection.selectedAgent);
    }

    // Step 3: Send message to selected agent
    const response = await api.chatWithAgent(
      activeAgent.id,
      content,
      messages
    );

    setMessages([...messages, { content, sender: 'user' }, response]);
  };

  return (
    <div className="chat-container">
      {/* Agent Selector */}
      <AgentSelector
        activeAgent={activeAgent}
        agents={availableAgents}
        onAgentSelect={setActiveAgent}
        intentDetection={intentDetection}
      />

      {/* Messages */}
      <MessageList messages={messages} />

      {/* Input */}
      <ChatInput onSend={handleSendMessage} />
    </div>
  );
};
```

## Challenges & Lessons Learned

### 1. Ambiguity Handling

Intent detection không phải lúc nào cũng chính xác. Chúng tôi học được:
- **Confidence thresholding**: Nếu confidence < 0.7, dùng General Agent để hỏi clarification
- **Context awareness**: Xem conversation history để hiểu intent tốt hơn
- **Explicit overrides**: Luôn cho phép người dùng chọn agent thủ công

### 2. Agent Switching Complexity

Khi chuyển agent giữa conversation:
- **Context loss**: Agent mới không có context của cuộc trò chuyện cũ
- **Solution**: Summarize conversation trước khi chuyển agent
- **UI feedback**: Hiển thị agent transition rõ ràng cho người dùng

### 3. Performance Optimization

Embedding-based intent detection có thể chậm:
- **Cache common patterns**: Cache intent results cho queries phổ biến
- **Batch processing**: Process multiple messages in batch khi có thể
- **Async streaming**: Stream response trong khi detect intent

### 4. Testing Strategy

Testing intent detection thực sự khó:
- **Golden dataset**: Tạo dataset với known intents và expected agents
- **A/B testing**: Deploy gradual rollout với metric tracking
- **User feedback**: Bắt người dùng rate agent selection accuracy

## Future Improvements
